# 搜索功能

Note System 提供基于 GitHub Code Search API 的全文搜索能力。

## 使用方式

### 搜索栏

在页面顶部右侧的搜索栏输入关键词，系统会自动防抖搜索（300ms 延迟），避免频繁 API 调用。

### 搜索结果

搜索结果以卡片列表展示，包含：

- 笔记标题
- 文件路径
- 匹配片段高亮

### 标签搜索

点击笔记中的标签（如 `#guide`）会跳转到标签搜索页面，自动搜索带有该标签的笔记。

> [!tip] 内联标签
> 在笔记正文中使用 `#标签名` 语法添加内联标签。例如本篇有：#搜索 #指南

## 搜索 API

```
GET /api/search?q=关键词
```

### 搜索范围

搜索限定在配置的笔记仓库和路径内。如果 `NOTES_PATH` 为空，搜索整个仓库。

### 缓存

搜索结果缓存 2 分钟（比笔记内容的 5 分钟短），以保持较新的搜索结果。

## 搜索技巧

> [!note] 提示
> GitHub Code Search 支持一些高级语法：

1. **精确匹配**：使用引号 `"exact phrase"`
2. **排除词**：使用减号 `-exclude`
3. **文件类型**：`extension:md`

## 与链接系统的关系

搜索功能与 Obsidian 链接系统互补：

- **搜索** — 发现你可能不知道的笔记
- **[[wikilinks]]** — 建立已知笔记之间的关联
- **反向链接** — 发现谁引用了你的笔记

链接语法详见 [[指南/Obsidian语法]]，API 文档详见 [[指南/API参考]]。

## 性能考量

搜索 API 调用 GitHub Search API，受到 GitHub 限流约束。高频搜索场景下的优化方案参考 [[指南/性能优化]]。
